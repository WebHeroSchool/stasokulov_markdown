## Пишите код понятно
Гораздо больше сил и времени уходит на поддержание кода, чем на его написание. Ваш код будут читать и пытаться понять другие люди. Очень важно, чтобы они могли разобраться в нем как можно проще и быстрее. Да и вы сами, через некоторое время, забудете, как работает ваш код. Все правила стиля, даже если они кажутся не удобными при написании, следует соблюдать ради будущей поддержки кода.

## Отступы
Если новая строка кода находится внутри логического блока, открытого предыдущей строкой, то пишите новую строку с отступом в 2 пробела.
#### Примеры
alert находится внутри логического блока, который начинается с if, поэтому он получает отступ в 2 пробела.
```
if (a>b) {
  alert('а больше b');
}
```
Переменные представляют собой отдельные логические блоки, поэтому пишутся одна над другой без отступа.
```
let a = 3;
let b = 8;
```
Плохой пример. Так читаемость кода ниже.
```
if (a>b) {
alert('а больше b');
}
```

## Фигурные скобки
Открывающая фигурная скобка пишется на той же строке, что предшествующий ей и связанный с ней код. После открывающей фигурной скобки идет перенос строки. Перед закрывающей фигурной скобкой, также делается перенос строки.
#### Примеры
Хороший пример
```
function sayGo() {
  alert('Вперед!');
}
```
Плохой пример
```
function sayGo() {alert('Вперед!');}
```

## Пробелы и операторы
- Унарные операторы пишутся слитно с операндами. Например, оператор декремента `i++`, оператор "не" `!true`.
- Бинарные операторы отделяются пробелами от операндов.
```
if (a > b) {
  return a + b;
}
```
- Так же как и тернарные.
```
a == true ? alert('"a" существует!') : alert('С "а" не все понятно')
```
## Пробелы и параметры
Параметры разделяются запятой и пробелом.
#### Примеры
Хороший пример. После а идет запятая и пробел.
```
let = raise = (a, b) => {
  a ** b;
}
```
Плохой пример.
```
let = raise = (a,b) => {
  a ** b;
}
```
## Пустая строка между логическими блоками
Отделяйте логические блоки пустой строкой. В данном случае мы отделяем объявленные переменные и функцию.
```
let a = 3;
let b = 9;

let sum = (a, b) => {
  return a + b;
}
```
## Шаблонные строки вместо конкатенации
Шаблонные строки - это блоее читабельное решение.
#### Примеры
Плохой пример.
```
function sayHi(name) {
  return 'How are you, ' + name + '?';
}
```
Хороший пример.
```
function sayHi(name) {
  return `How are you, ${name}?`;
}
```
## Имена функций
Имена функций должны быть глаголами, а имена переменных - существительными. Из их названий должно быть понятно что они делают или содержат.
#### Примеры
Хорошие примеры.
```
function divide(a, b) {
  return a / b;

let name = 'Вася';
```
Плохие примеры.
```
function calculate(a, b) {
  return a / b;

let a = 'Вася';
```
## camelCase
Если имя функции или переменной состоит из нескольких слов, то следует соединять слова в стиле camelCase.
#### Примеры
Хороший пример.
```
let carColor = 'red';

function getNameUser() {
  return user.name;
}
```
## Цепное присвоение значений переменным
Не следует использовать цепное присвоение переменных внутри функций, т.к. часть из них могут оказаться в глобальной области видимости.
#### Примеры
Плохой пример. Переменная а останется локальной, а b и c  - станут глобальными.
```
(function foo() {
  let a = b = c = 1;
}

console.log(a); // переменная не существует
console.log(b); // 1
console.log(c); // 1
```
Хороший пример. Все переменные остаются локальными.
```
(function foo() {
  let a = 1;
  let b = 1;
  let c = 1;
}

console.log(a); // переменная не существует
console.log(b); // переменная не существует
console.log(c); // переменная не существует
```
